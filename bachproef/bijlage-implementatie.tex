\section{Orb agent: shellscripts voor uitrol}
\label{sec:bijlage-orb-scripts}

\subsection{Script voor SNMP discovery}

\begin{listing}[H]
  \caption[Shellscript voor SNMP-discovery uitrol]{Volledig shellscript voor het genereren van \texttt{agent.yaml} voor SNMP discovery (\S\ref{sec:schaalbaarheid})}
  \label{lst:orb-snmp-script}
  \begin{minted}[fontsize=\small]{bash}
#!/usr/bin/env bash
set -e

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
cd "$SCRIPT_DIR"

mkdir -p "$SCRIPT_DIR/snmp-extensions"

CREDS_FILE="/opt/diode/oauth2/client/client-credentials.json"
ALT_CREDS_FILE="/vagrant/diode/oauth2/client/client-credentials.json"

if [ ! -f "$CREDS_FILE" ]; then
  if [ -f "$ALT_CREDS_FILE" ]; then
    CREDS_FILE="$ALT_CREDS_FILE"
  else
    echo "Fout: $CREDS_FILE niet gevonden."
    exit 1
  fi
fi

if ! command -v envsubst >/dev/null 2>&1; then
  echo "envsubst niet gevonden. Installeer met: sudo apt-get install -y gettext-base"
  exit 1
fi

export DIODE_CLIENT_ID="diode-ingest"
export DIODE_CLIENT_SECRET=$(sudo jq -r '.[] | select(.client_id == "diode-ingest") | .client_secret' "$CREDS_FILE")

if [ -z "$DIODE_CLIENT_SECRET" ]; then
  echo "Fout: kon diode-ingest secret niet ophalen."
  exit 1
fi

export DEVICE_USERNAME="user"
export DEVICE_PASSWORD="pass"

envsubst '${DIODE_CLIENT_ID} ${DIODE_CLIENT_SECRET} ${DEVICE_USERNAME} ${DEVICE_PASSWORD}' < agent.yaml.snmp > agent.yaml
echo "agent.yaml gegenereerd met credentials."

exec sudo docker run --rm --network host \
  -v "$SCRIPT_DIR:/opt/orb" \
  -e PIP_BREAK_SYSTEM_PACKAGES=1 \
  netboxlabs/orb-agent:latest run -c /opt/orb/agent.yaml
  \end{minted}
\end{listing}

\section{Volledige uitrolprocedure Diode-server}
\label{sec:bijlage-diode-uitrol}

Deze bijlage documenteert de volledige uitrol van de Diode-server op de Citymesh-stagingomgeving, zoals samengevat in \S\ref{sec:uitrol-diode}. Concrete secret-waarden zijn vervangen door placeholders.

\subsection{Prerequisites op de hostmachine}

Vooraleer de Diode-stack kan opgestart worden, moeten op de hostmachine de volgende afhankelijkheden aanwezig zijn:

\begin{itemize}
  \item NetBox Community~4.2.3 of nieuwer (de gebruikte stagingomgeving draait op versie~4.4.10);
  \item Docker~27.0.3 of nieuwer;
  \item Bash~4.x of nieuwer;
  \item \texttt{jq} (om OAuth2-credentials uit \texttt{client-credentials.json} te parsen);
  \item een gebruikersaccount dat in de \texttt{docker}-groep zit, zodat \texttt{docker compose} zonder extra \texttt{sudo}-stappen kan draaien.
\end{itemize}

\begin{listing}[H]
  \caption[Verificatie van de hostprerequisites]{Verificatie van de hostprerequisites vóór de Diode-installatie}
  \label{lst:diode-prereq-verify}
  \begin{minted}[fontsize=\small]{bash}
echo "Bash version: $(bash --version | head -n 1)"
echo "Jq version: $(jq --version)"
docker --version
groups $USER | grep -q "docker" \
  && echo "Docker permissions: OK" \
  || echo "Docker permissions: log out/in vereist"
  \end{minted}
\end{listing}

\subsection{Diode-server: \texttt{quickstart.sh}}

De Diode-stack zelf wordt opgezet met het door NetBox Labs aangeleverde \texttt{quickstart.sh}-script. Dat script genereert een \texttt{docker-compose.yaml}, een \texttt{nginx}-configuratie, een \texttt{.env}-bestand en het centrale \texttt{client-credentials.json}-bestand met OAuth2-credentials voor twee logische clients (\texttt{diode-ingest} voor de Orb agent en \texttt{netbox-to-diode} voor de NetBox-plugin).

\begin{listing}[H]
  \caption[Uitrol van de Diode-server via \texttt{quickstart.sh}]{Uitrol van de Diode-server via het door NetBox Labs aangeleverde \texttt{quickstart.sh}-script (\S\ref{sec:uitrol-diode})}
  \label{lst:diode-quickstart}
  \begin{minted}[fontsize=\small]{bash}
sudo mkdir -p /opt/diode
cd /opt/diode

sudo curl -sSfLo quickstart.sh \
  https://raw.githubusercontent.com/netboxlabs/diode/release/diode-server/docker/scripts/quickstart.sh
sudo chmod +x quickstart.sh

./quickstart.sh https://<netbox-host>

docker compose up -d

docker compose ps
  \end{minted}
\end{listing}

Na een succesvolle run draaien zeven containers samen: \texttt{nginx} (ingress), \texttt{diode-auth}, \texttt{diode-ingester}, een Ory Hydra-instantie voor OAuth2, \texttt{diode-reconciler}, \texttt{postgres} (persistente state) en \texttt{redis} (queueing).
