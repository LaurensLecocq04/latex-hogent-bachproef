\maketitle

\begin{abstract}
Deze bachelorproef onderzoekt hoe automatische netwerkdetectie de actualiteit en juistheid van netwerkdocumentatie in NetBox kan verbeteren. NetBox Discovery met Diode werd gebruikt als basis voor inventarisdata. Voor onderdelen die niet volledig automatisch landen in NetBox, zoals LLDP-kabelrelaties, werd een aanvullende \texttt{NetworkTool}-workflow via de NetBox API toegepast. De resultaten tonen een duidelijke verbetering van documentatiekwaliteit in twee praktijkcases.
\end{abstract}

\begin{multicols}{2}

\section{Context en probleem}
Citymesh beheert dynamische netwerkinfrastructuren. In zo'n omgeving raakt documentatie snel verouderd wanneer wijzigingen niet consequent worden doorgevoerd.

Vastgestelde knelpunten:
\begin{itemize}
  \item handmatige CSV-imports zijn tijdrovend en foutgevoelig;
  \item vervangingen van toestellen worden niet altijd bijgewerkt;
  \item topologie- en kabelrelaties zijn vaak onvolledig.
\end{itemize}

\textbf{Onderzoeksvraag}\\
Hoe kan automatische netwerkdetectie bijdragen aan up-to-date netwerkdocumentatie en een betrouwbare \emph{single source of truth} (SSOT) in NetBox?

\section{Methodologie}
\begin{itemize}
  \item \textbf{Fase 1 - Requirementsanalyse:} inventariseren van pijnpunten in de huidige werkwijze.
  \item \textbf{Fase 2 - Toolkeuze:} vergelijking van NetBox Discovery en Slurp'it op haalbaarheid en bruikbaarheid.
  \item \textbf{Fase 3 - Baseline:} nulmeting op bestaande NetBox-documentatie per site.
  \item \textbf{Fase 4 - Implementatie:} configuratie van Diode + integratie met \texttt{NetworkTool}.
  \item \textbf{Fase 5 - Evaluatie:} voor/na-analyse op datakwaliteit en kabelrelaties.
  \item \textbf{Fase 6 - Analyse:} interpretatie van resultaten, beperkingen en aanbevelingen.
\end{itemize}

\section{Technische aanpak}
\textbf{Basis:} NetBox Discovery met Diode voor device- en inventarisdetectie.\\
\textbf{Aanvulling:} \texttt{NetworkTool} met:
\begin{itemize}
  \item dry-run (\emph{diff}) voor gecontroleerde wijzigingsanalyse;
  \item live push via NetBox API na validatie;
  \item LLDP-gebaseerde correctie en aanvulling van kabelrelaties.
\end{itemize}

\section{Resultaten}
\textbf{Case 1: Ridefort}
\begin{itemize}
  \item baseline: 3/6 AP-kabelrelaties correct;
  \item na discovery + \texttt{NetworkTool}: 6/6 AP-kabelrelaties correct;
  \item AP-naamgeving en toesteltypes zijn gecorrigeerd.
\end{itemize}

\textbf{Case 2: Deleers Go4tech}
\begin{itemize}
  \item siteomvang: 11 switches, 1 router, 1 Fortinet firewall, 48 AP's;
  \item bij herdiscovering werden 3 switches met fout type gedetecteerd, ondanks recente documentatie.
\end{itemize}

\textbf{Kerninzicht}\\
Discovery corrigeert niet alleen historische fouten, maar werkt ook als kwaliteitscontrole op recente documentatie.

\section{Conclusie}
NetBox Discovery met Diode is een haalbare en effectieve basis om inventarisdata in NetBox actueel te houden. Voor volledige topologie- en kabelkwaliteit is aanvullende automatisering nodig. In deze casestudy werd dat gerealiseerd met \texttt{NetworkTool} en de NetBox API.

\textbf{Belangrijkste beperking:} serienummers van via SNMP ontdekte AP's worden niet automatisch ingevuld.

\section{Aanbevelingen}
\begin{itemize}
  \item gebruik periodieke discovery als standaard kwaliteitscontrole;
  \item behoud een dry-run stap voor elke live synchronisatie;
  \item veranker cleanup en naamgevingsregels in operationele processen;
  \item voorzie een aparte datastroom voor serienummers buiten SNMP.
\end{itemize}

\end{multicols}

\begin{abstract}
Deze bachelorproef onderzoekt hoe automatische netwerkdetectie kan bijdragen aan actuele netwerkdocumentatie en een betrouwbare \emph{single source of truth} (SSOT) binnen Citymesh. NetBox Discovery met Diode werd geïmplementeerd als basis, aangevuld met \texttt{NetworkTool} voor datatypes die niet volledig automatisch landen in NetBox (met focus op kabelrelaties op basis van LLDP-neighbors). De casestudy toont dat discovery fouten sneller zichtbaar maakt en documentatiekwaliteit verhoogt.
\end{abstract}

\begin{multicols}{2}

\section{Probleemstelling}

In dynamische infrastructuren veroudert netwerkdocumentatie snel door wijzigingen die niet altijd correct of tijdig worden bijgewerkt.

Belangrijkste knelpunten bij Citymesh:
\begin{itemize}
  \item handmatige CSV-workflows voor inventarisupdates;
  \item verouderde records na infrastructuurwijzigingen;
  \item onvolledige of foutieve topologie- en kabelinformatie.
\end{itemize}

\textbf{Centrale onderzoeksvraag:}
hoe kan automatische netwerkdetectie bijdragen aan het up-to-date houden van netwerkdocumentatie en aan een betrouwbare SSOT?

\section{Aanpak}

\begin{itemize}
  \item \textbf{Toolselectie:} NetBox Discovery (met Diode) als primaire oplossing; Slurp'it als referentie, maar financieel minder haalbaar op schaal.
  \item \textbf{Implementatie:} Orb-agent configuraties voor SNMP/device discovery in combinatie met NetBox.
  \item \textbf{Aanvulling:} modulaire \texttt{NetworkTool} met:
  \begin{itemize}
    \item dry-run analyse (\emph{diff}) voor gecontroleerde wijzigingen;
    \item live synchronisatie via NetBox API;
    \item LLDP-gebaseerde correctie van kabelrelaties.
  \end{itemize}
\end{itemize}

\section{Casestudy en resultaten}

\textbf{Case 1: Ridefort}
\begin{itemize}
  \item Baseline: 3/6 AP-kabelrelaties correct.
  \item Na discovery + \texttt{NetworkTool}: 6/6 kabelrelaties correct (LLDP switch), AP-namen correct.
\end{itemize}

\textbf{Case 2: Deleers Go4tech}
\begin{itemize}
  \item Omvang: 11 switches, 1 router, 1 Fortinet firewall, 48 AP's.
  \item Ook bij recente documentatie werden 3 switches met fout type gedetecteerd na herdiscovering.
\end{itemize}

\textbf{Belangrijk inzicht:}
discovery corrigeert niet alleen oude fouten, maar werkt ook als continue kwaliteitscontrole op recente documentatie.

\section{Conclusie}

NetBox Discovery met Diode is een sterke basis om inventarisdata automatisch actueel te houden.
Voor volledige topologie en bekabeling is aanvullende automatisering nodig; in deze casestudy werd dat opgelost met \texttt{NetworkTool} en NetBox API.

Resterende beperking: serienummers van via SNMP ontdekte AP's worden niet automatisch gevuld.

\section{Aanbevelingen}

\begin{itemize}
  \item Zet discovery periodiek in als standaard kwaliteitscontrole.
  \item Gebruik dry-run + live push om foutieve updates te vermijden.
  \item Neem cleanup en naming governance op in het wijzigingsproces.
  \item Voorzie een alternatieve datastroom voor serienummers buiten SNMP.
\end{itemize}

\section{Toekomstig werk}

\begin{itemize}
  \item Verbreden van evaluatie naar meer sites en langere meetperiodes.
  \item Verdere automatisering van serienummerverzameling.
  \item Uitbouwen van rapportering op documentatiekwaliteit (voor/na indicatoren per site).
\end{itemize}

\end{multicols}
